# DFA 操作流程說明：Delete、Merge 與 Delta

## Delete 流程

### 1. 定義目標
- **目標**：刪除 DFA 中的某些狀態以簡化結構。
- **條件**：確保刪除後的 DFA 仍然能正確區分接受與拒絕的輸入。

### 2. 流程步驟
1. **遍歷狀態**：刪除每個狀態。
2. **重新指向**：原本指向該狀態的 incoming transitions 重新導向其 outgoing transitions；若無對應 outgoing transition，則改為自迴圈
2. **輸出結果**：返回所有生成的新 DFA 列表。

### 3. 注意事項
- 保護關鍵狀態（例如，初始狀態或接受狀態），避免刪除後破壞 DFA 的語義或使 DFA 失去接受能力
- 使用簽名（signature）去重，避免生成重複的 DFA。

---

## Merge 流程

### 1. 定義目標
- **目標**：合併 DFA 中的兩個狀態以簡化結構。
- **條件**：優先選擇在訓練資料上表現相似的狀態對。

### 2. 流程步驟
1. **計算標籤分佈**：根據訓練資料最終到達的狀態，統計每個狀態對正負標籤樣本的覆蓋分佈。
2. **選擇合併對**：選擇具有相似標籤分佈的狀態對，優先合併行為一致的狀態。
3. **執行合併**：合併選定的狀態對。
4. **輸出結果**：返回所有生成的新 DFA 列表。

### 3. 注意事項
- 避免將接受性不同的狀態直接合併，並優先考慮在訓練資料上行為相似的狀態對，以降低語義破壞風險。
- 控制合併對的數量，避免過多的計算開銷。
- 使用簽名（signature）去重，避免生成重複的 DFA。

---

## Delta 流程

### 1. 定義目標
- **目標**：通過修改 DFA 的轉移邊來改進準確度。
- **方法**：使用 CXP 分析找出問題邊，並改接到更合適的狀態。

### 2. 流程步驟

1. 從誤分類路徑中取 batch size 條路徑
2. 統計這些錯誤路徑上的 edge frequency
3. 找出最常出現的一條錯誤 edge
4. 從包含這條 edge 的路徑中，選最短的一條 path
5. 只對這 1 條 path 算 CXP
6. 從 shortest CXP 對應 edges 中，只選最靠近決策點的一條
7. 對這 1 條 edge，嘗試改接到所有其他 states，生成候選 DFA

edge frequency 用來找共同錯誤地區，選一條代表路徑，找出造成分類錯誤的位置 (cxp)
